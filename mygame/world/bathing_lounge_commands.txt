-- STEP 10: Mimic mechanic (future, standalone system) --

ACTIVATION
  When 'start shower' fires, roll against mimic_chance (default 30%).
  On success: delay 20-30 seconds (shower runs normally first), then
  mimic activates — script text rises on the glass walls, room locks
  (exit returns a flavor message), mimic_active = True.
  On miss: normal shower, no mimic, no lock.

SCENE PROGRESSION
  Scenes stored in world/mimic_scenes.yaml, organized by phase:
    opening  — 1 scene, sets the mood, ~30-60 seconds in
    rising   — pool of scenes, 2-3 drawn randomly, ~2 min apart
    peak     — pool of scenes, 1-2 drawn, escalating
    climax   — 1 scene, the resolution beat
  Each phase draws randomly from its pool. Scenes are tagged by
  manifest type so the pool can be extended without code changes.

MANIFEST TYPES (tags for scene YAML entries)
  The mimic can produce any of the following based on the scene:
    cock      — any shape, size, species, or configuration
    orifice   — same range; can be independent of a body
    plug      — inserted/filling shapes, passive or active
    tentacle  — prehensile, multiple, independent
    combined  — two or more types simultaneously
  Tags allow future filtering (e.g. room flag to exclude certain types)
  but default behavior draws from all categories freely.

CLIMAX AND RELEASE
  After climax scene fires: 60-90 second cooldown, then release message,
  shower turns off, lock drops, steam begins dissipating.
  mimic_active = False, mimic_phase = None, locked = False.

AMBIENT (separate from mimic, fires whenever shower is running)
  A repeat script with randomized interval: 300-600 seconds (5-10 min).
  Small sensory lines — steam, warmth, rune shimmer, drain sound, heat
  on skin. Quiet background texture. Does not fire during mimic sequence
  (mimic scenes replace ambient for that duration).
